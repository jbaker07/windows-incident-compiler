Purpose: Complete pipeline documentation from capture to UI display

================================================================================
OVERVIEW
================================================================================

Data Pipeline:
  capture_windows_rotating.exe -> JSONL segments -> edr-locald.exe -> 
    facts extraction -> signal detection -> workbench.db -> 
      locint HTTP API -> UI polling

Process Boundaries:
  1. capture_windows_rotating.exe - ETW/Sysmon capture, writes JSONL
  2. edr-locald.exe - Processes segments, extracts facts, detects signals
  3. locint.exe / edr-server.exe - HTTP server, serves UI, manages runs

================================================================================
CAPTURE LAYER
================================================================================

Binary: capture_windows_rotating.exe
Source: crates/agent-windows/

Input:
  - ETW providers (Microsoft-Windows-Kernel-Process, etc.)
  - Sysmon events (Event IDs 1, 3, 7, 8, 10, 11, 13, etc.)

Output:
  - {data_dir}/runs/run_{id}/segments/*.jsonl
  - Rotating files, ~10MB each

Spawn Location:
  locint.rs::run_start_handler() line 217
  RunController::start() in run_control.rs line 182

================================================================================
PROCESSING LAYER (LOCALD)
================================================================================

Binary: edr-locald.exe
Source: crates/locald/

Pipeline (pipeline.rs):
  1. SegmentWatcher - monitors segments/ directory
  2. EventParser - parses JSONL events
  3. FactExtractor - extracts structured facts from events
  4. SignalOrchestrator - routes facts to platform-specific detectors
  5. SignalPersistence - writes signals to workbench.db

Fact Types Extracted:
  - Exec (process execution)
  - FileWrite, FileDelete
  - RegWrite, RegDelete
  - NetConnect
  - Auth (logon events)
  - PersistArtifact

Signal Detection:
  - WindowsSignalEngine (os/windows/signal_engine.rs)
  - Hardcoded detectors: LSASSAccessSuspicious, WmiPersistenceConfirmed, etc.
  - PlaybookManager for YAML-based rules
